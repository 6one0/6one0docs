    # 什么是Minecraft?

    Minecraft (opens new window)是一款基于Java的游戏，它是由Mojang于2009年开发的。 《我的世界》（Minecraft）是一款沙盒类电子游戏，开创者为马库斯·阿列克谢·泊松（Notch）。游戏由Mojang Studios维护，现隶属于微软Xbox游戏工作室。游戏最初于2009年5月17日作为Classic版本发布，并于2011年11月18日发布Java正式版。我的世界的游戏平台囊括桌面设备、移动设备和游戏主机。中国版现由网易游戏代理，于2016年5月20日在中国大陆运营。
    自开创伊始到延斯·伯根斯坦（Jeb）加入并负责开发之前，我的世界几乎全部的开发工作由Notch完成。游戏音乐由丹尼尔·罗森菲尔德（C418）和莉娜·雷恩（Lena Raine）创作；克里斯托弗·泽特斯特兰绘制了游戏中的画。
    该游戏以玩家在一个充满着方块的三维空间中自由地创造和破坏不同种类的方块为主题。玩家在游戏中可以在单人或多人模式中通过摧毁或创造精妙绝伦的建筑物和艺术，或者收集物品探索地图以完成游戏的成就（进度）。玩家也可以尝试在创造模式下(打开作弊)红石电路和指令等玩法。
    ——摘自百度百科 (opens new window)

    # 什么是 Java 版/基岩版？

    如果您不是云玩家，那您应该听过手机版、电脑版两个称呼，这分别是基岩版（Bedrock Edition，BE）、 Java 版（Java Edition，JE）的俗称。
    您可能有些生气：“为什么不直接用手机版/电脑版的称呼呢？多好记！这两个该死的名称害得我思考了半天！” 携带版（Pocket Edition，PE）在历史上的确存在过，但在1.2 以后（含），其被更名为基岩版。原因可能是因为从此以后其开始适应诸如电脑等其他游戏平台了（毕竟将一个可以在电脑上玩的游戏叫做携带版，总觉得不合适）。
    电脑版（PC）与手机版（携带版别名）相对，一般指 Java 版。不过电脑版与手机版都不是官方的称呼，且其具有一定的误导性（毕竟 Java 版和基岩版都可以在电脑上运行）。
    综上所述，不建议使用手机版、电脑版这种称呼，而应该用基岩版或 Java 版，尤其是技术玩家。

    PS：在基岩版 1.2 以前的版本需使用携带版（PE）。

    PSS：Windows 10 版、教育版隶属于基岩版。

    # MinecraftJE依赖于Java!

    MinecraftJE依赖于Java，因此你需要下载并安装Java (opens new window)。

    WARNING

    不同的MC版本需要不同的Java!
    如：1.0 - 1.11.x 可以使用 Java 6 和 Java 7，但推荐使用 Java 8。
    1.12(17w13a) - 1.16.5(1.17-21w18a) 需要使用 Java 8。
    1.17(21w19a) - 1.17.1 需要使用 Java 16。
    1.18(1.18-pre2) 及以上版本需要使用 Java 17。

    # Java

    Java 是Sun公司发明的，2006年末Sun公司宣传将开放Java源代码。
    2009 年Oracle公司收购了 Sun 公司，而 Sun 公司在被收购的几天前发布了 OpenJDK，即开源版本的 JDK（源代码）。
    收购后Oracle公司推出了OracleJDK 由于正统且免费性能好，于是大家都开始用OracleJDK。
    但后来由于Oracle公司对于Java社群的不友善和商业化，逐渐催生了许多基于OpenJDK的第三方 JDK。 在下载Java时你可能会看到JDK、JER、JVM等多个名称。

    # JDK?

    JDK是Java开发工具包，也就是说这个适用于开发Java程序。
    从技术上来说，OpenJDK 仅提供源代码（现在也直接提供可执行文件）所以你不能直接使用OpenJDK来运行Java应用程序，而必须自己构建 OpenJDK 源代码来获得可执行文件，或者找一个会为你构建的厂商（即发行版）。
    只要符合规范，不同 JDK 发行版应该均可互相兼容。但不同的 JDK 细节表现、性能、支持的系统/内存等各有差别，也可能有不同的功能，你可以查看测评或自行测试来决定使用哪一个。
    目前流行的 JDK 发行版有： OracleJDK (opens new window)、OpenJDK (opens new window)、Zulu (opens new window)、Dragonwell (opens new window)、Liberica (opens new window)、RedHatOpenJDK (opens new window)、Microsoft (opens new window)、AdoptOpenJDK (opens new window)（提供 Openj9）、GraalVM (opens new window)（其实不是 JDK，但它有 Jvm）等。
    目前我们最推荐使用JDK来运行MinecraftJE。
    使用：Zulu (opens new window)最推荐，其占用内存少，对MCJE可以提高fps，运行速度等、Dragonwell (opens new window)、GraalVM (opens new window) 这些应该比原版 OpenJDK 有着更好的效果。
    也有人说用微软自家的 Microsoft Build of OpenJDK (opens new window) 也许会有神秘加成，此外本家的 OracleJDK (opens new window) 也是经典的选择。

    # JRE?

    JRE是Java运行环境，也就是说这个适用于运行Java程序（比如MC）。因此我们次要推荐使用JER运行MinecraftJE。

    # JVM?

    JVM是Java虚拟机，也就是说这个也适用于运行Java程序。Java虚拟机屏蔽了与具体操作系统平台相关的信息，使得Java程序只需生成在Java虚拟机上运行的目标代码（字节码），就可以在多种平台上不加修改地运行。这是运行MinecraftJE的最后手段。
